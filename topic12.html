Developer History & Fundamentals

In the early 2000s applications were built with technologies HTML, CSS, JavaScript + JQuery. JQuery was a library that allowed JavaScript to be written in a simple clean way, allowed developers not to worry so much about working in one browser than the other. It created this environment for JavaScript to flourish. Nowadays JQuery is not used very often and is an outdated technology. On the Back End we had the LAMP stack, that stands for L-Linux, A-Apache, M-MySQL and P-PHP. In those days back-end developers used PHP to write files that allowed us to have logic on the servers. Apache is a software that really efficiently serves up files. Nowadays in order for Full Stack Developers to build clean, fast and bug free web applications they need to know: Front End Developer - HTML5, CSS3, JavaScript + React ; Back End Developer - Node Server, Node.js, Express.js, Database, PostgrSQL, MongoDB.

Another very important aspect of being a Web Developer is the fact that we have multiple web browsers (Google Chrome, Mozilla Firefox, Apple's Safari etc.). When we send files over each of these browsers read the HTML, CSS and JavaScript and layout the web page for us, but because each of them is owned by a company and is a different browser so that means they all have to agree on how they're gonna read the HTML, CSS and JavaScript. If they don't all agree then we need to create different files specific for each browser because each one of them has a different idea of what HTML, CSS and JavaScript is. Some of the biggest problems are that developers need to figure out whenever they send their files to the web browser it all looks the same and browsers agree on what code works in each of these files. Also developers have to make sure that their websites look good on all sorts of different screens.
